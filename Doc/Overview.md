# Overview

This repository provides the processing pipeline used to reconstruct **historical water distribution calendars**
for Tegucigalpa, Honduras, supporting SDG-6 monitoring and longitudinal urban services research.

## Scope

The pipeline processes calendar schedule inputs (structured or OCR-derived) and produces:

- standardized tabular outputs (calendar-level and aggregated metrics)
- QA/QC reports for identifier and mapping integrity
- optional geospatial outputs linking water schedules to neighborhood units

This GitHub repository contains **pipeline code and documentation only**.
Full datasets generated by this pipeline are archived on **Zenodo** (DOI added upon release).

## Processing stages (conceptual)

1. Schedule parsing and text normalization
2. Extraction of water availability metrics (hours and/or days)
3. Matching to water distribution identifiers (`wc_uid`, `group_uid`)
4. Validation against a canonical catalog (water sector / source / distribution unit)
5. QA reporting for missing or inconsistent identifiers
6. Merge/aggregation to produce standardized outputs
7. Optional GIS export (GeoPackage or equivalent)

## Outputs (examples)

Typical outputs include:

- calendar-level hours dataset (e.g., `all_horas.csv`)
- calendar-level days dataset (optional)
- QA reports (CSV)
- merged dataset enriched with catalog attributes
- geospatial layer(s) for mapping and integration with SDG-11 neighborhood units

## Intended use

The pipeline is designed for:

- reproducible reconstruction of historical service schedules
- transparent QA/QC and traceable processing
- integration with geospatial research on basic services and infrastructure
